=== 전용 목적 설명 (1000자 이내) ===

유니버설 디코더는 다양한 형식의 인코딩된 텍스트를 디코딩하는 전용 도구입니다. 사용자가 입력한 텍스트를 로컬에서 분석하여 URL, HTML, Base64, JWT, Hex, ROT13, GZIP 등 10가지 이상의 인코딩 형식을 자동으로 감지하고 디코딩합니다. 모든 처리는 사용자의 브라우저 내에서 로컬로 수행되며, 외부 서버로 데이터를 전송하지 않습니다. 이 확장 프로그램은 개발자, 보안 연구원, 데이터 분석가가 인코딩된 데이터를 빠르고 쉽게 분석할 수 있도록 돕는 것이 유일한 목적입니다.


=== activeTab 사용 근거 (1000자 이내) ===

activeTab 권한은 사용자가 현재 웹 페이지에서 선택한 인코딩된 텍스트를 빠르게 디코딩할 수 있도록 하기 위해 필요합니다. 개발자나 보안 연구원이 웹 페이지에서 발견한 인코딩된 문자열(예: API 응답, JWT 토큰, 인코딩된 URL 등)을 선택하고 확장 프로그램을 통해 즉시 디코딩할 수 있습니다. 이 권한은 사용자가 확장 프로그램 아이콘을 클릭했을 때만 활성화되며, 사용자가 명시적으로 요청하지 않는 한 웹 페이지에 접근하지 않습니다. 모든 디코딩 작업은 로컬에서 수행되며, 데이터가 외부로 전송되지 않습니다.


=== scripting 사용 근거 (1000자 이내) ===

scripting 권한은 웹 페이지에서 사용자가 선택한 텍스트나 인코딩된 데이터를 추출하기 위해 필요합니다. 사용자가 웹 페이지에서 인코딩된 문자열을 발견했을 때, 해당 텍스트를 수동으로 복사하지 않고도 확장 프로그램이 선택된 텍스트를 자동으로 가져와 디코딩할 수 있도록 합니다. 이는 특히 개발자 도구나 네트워크 탭에서 발견한 인코딩된 응답 데이터를 빠르게 분석하는 데 유용합니다. 이 권한은 사용자가 확장 프로그램을 명시적으로 사용할 때만 실행되며, 웹 페이지의 콘텐츠를 수정하거나 변경하지 않습니다. 모든 데이터 처리는 로컬에서 이루어지며 외부로 전송되지 않습니다.


=== storage 사용 근거 (1000자 이내) ===

storage 권한은 사용자의 선호 설정을 저장하기 위해 필요합니다. 구체적으로 다음과 같은 사용자 설정을 로컬 저장소에 저장합니다:

1. 다크모드 설정: 사용자가 선택한 다크모드/라이트모드 테마 설정을 저장하여 다음 사용 시에도 동일한 테마가 적용되도록 합니다.

2. 디코더 타입 설정: 사용자가 자주 사용하는 디코딩 타입(예: Base64, JWT 등)을 저장하여 매번 선택할 필요 없이 자동으로 적용되도록 합니다.

이러한 설정은 사용자 경험을 향상시키기 위한 것이며, 모든 데이터는 사용자의 로컬 브라우저에만 저장됩니다. 외부 서버로 전송되지 않으며, 개인정보를 수집하지 않습니다.


=== 호스트 권한 사용 근거 (1000자 이내) ===

호스트 권한(http://*/*, https://*/*)은 확장 프로그램이 모든 웹사이트에서 작동할 수 있도록 하기 위해 필요합니다. 개발자와 보안 연구원은 다양한 웹사이트에서 인코딩된 데이터를 발견할 수 있으며, 어떤 웹사이트에서든 일관된 방식으로 디코딩 기능을 사용할 수 있어야 합니다. 이 권한은 사용자가 확장 프로그램을 명시적으로 사용할 때만 활성화되며, 웹 페이지의 콘텐츠를 수정하거나 사용자의 브라우징 활동을 추적하지 않습니다. 모든 디코딩 작업은 로컬에서 수행되며, 웹 페이지의 데이터가 외부 서버로 전송되지 않습니다.
